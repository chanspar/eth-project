# 🏁 Ethereum Pipeline Data Integrity & Recovery Report

이 문서는 2026-05-02 데이터의 **심각한 유실 현상(99% 누락)**을 진단하고, 이를 **100% 복구**함과 동시에 향후 재발 방지를 위한 **품질 게이트(Quality Gate)**를 구축한 과정을 기록합니다.

---

## 1. 🔍 문제 진단 (Root Cause Analysis)

### 초기 증상
- **Bronze**: `transactions` (317만 건) vs `receipts` (2.1만 건)
- **Silver**: `txn_enriched` (2.1만 건)
- **결과**: `txn_enriched` 가공 시 `Inner Join`으로 인해 영수증이 없는 **315만 건의 트랜잭션이 증발**.

### 원인 파악
- **익스트랙터 설계**: 비용 절감을 위해 **1 ETH 이상의 고래(Whale)** 트랜잭션에 대해서만 영수증을 수집하도록 설정되어 있었음.
- **가공 로직의 경직성**: Silver 레이어 조인 시 `Inner Join`을 사용하여, 영수증이 없는 모든 일반 거래 및 토큰 거래가 삭제됨.

---

## 2. 🛠️ 해결 및 복구 작업 (The Recovery)

### [STEP 1] Silver 가공 로직 유연화 (`LEFT JOIN` 적용)
- `txn_enriched.py`와 `token_flow.py`의 조인 방식을 `Inner`에서 **`Left`**로 변경.
- 영수증 정보가 없더라도 트랜잭션의 기본 메타데이터와 토큰 전송 내역을 모두 보존하도록 수정.

### [STEP 2] 데이터 전수 재가공
- 수정된 로직으로 5월 2일 데이터를 다시 빌드하여 **317만 건의 트랜잭션과 245만 건의 토큰 흐름**을 전량 확보.
